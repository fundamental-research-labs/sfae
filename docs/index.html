Your agent makes API calls.
It never sees your keys.

Credentials stay in your secret storage. Agents use placeholders. Secrets never hit the context window.

Features

  • Keychain-native storage
    macOS Keychain, Windows Credential Manager, Linux Secret Service. Not env vars.
  • All sorts of credentials
    Basic Auth, API Key, OAuth 2.0, and more.
  • Communication protocols
    HTTP by default, plus Postgres and Redis with protocol-aware execution.

Roadmap

  • MySQL / MariaDB
    Native queries for MySQL-compatible production databases.
  • MongoDB
    Document and aggregation workflows with credential-safe connection strings.
  • Microsoft SQL Server / TDS
    Enterprise SQL Server and Azure SQL access through the trusted runtime.
  • ClickHouse
    Analytics queries without exposing database credentials to agents.

Install

  • Just install the SFAE skill
    Recommended

    Adds the SFAE skill to your project. It can install the CLI when needed.

    curl -fsSL https://sfae.io/install-skill.sh | sh

    Install for only one agent with:
    curl -fsSL https://sfae.io/install-skill.sh | sh -s -- --codex

  • Skill plus CLI

    Install the skill and immediately install the CLI through Homebrew, npm, or the direct release installer.

    curl -fsSL https://sfae.io/install-skill.sh | sh -s -- --install-cli
  • CLI only

    Use Homebrew first. npm installs the same native CLI through the published package; the direct installer is the fallback.

    Homebrew
    brew install fundamental-research-labs/tap/sfae
    npm
    npm install -g @fundamental-research-labs/sfae
    Direct
    curl -fsSL https://sfae.io/install.sh | sh

How it works

  1. Check

    Agent reads the service's official API/auth docs and checks what credentials exist for a domain.

    sfae credentials github.com
  2. Prompt

    Human is offered to provide credentials through a web form.

    sfae prompt github.com --spec '{
  "help_url": "https://github.com/settings/tokens",
  "fields": ["ACCESS_TOKEN"]
}'
  3. Request

    Agent makes calls with placeholders. SFAE resolves them.

    sfae request GET "https://api.github.com/user" \
  -H "Authorization: Bearer {ACCESS_TOKEN}" \
  -H "User-Agent: sfae"
