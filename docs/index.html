Your agent makes API calls.
It never sees your keys.

Credentials stay in Passwords on macOS. Agents use placeholders. Secrets never hit the context window.

Features

  • Keychain-native storage
    macOS Keychain, Windows Credential Manager, Linux Secret Service. Not env vars.
  • All sorts of credentials
    Basic Auth, API Key, OAuth 2.0, and more.
  • Communication protocols
    HTTP today; Postgres and other protocols are planned.

Install

  • Just install the SFAE skill
    Recommended

    Adds the agent skill to the current Codex project. The skill carries its own CLI installer for requests that need SFAE.

    curl -fsSL https://sfae.io/install-skill.sh | sh -s -- --codex
  • Skill plus CLI

    Install the skill and immediately install the CLI through Homebrew, npm, or the direct release installer.

    curl -fsSL https://sfae.io/install-skill.sh | sh -s -- --codex --install-cli
  • CLI only

    Use Homebrew first. npm installs the same native CLI through the published package; the direct installer is the fallback.

    brew install fundamental-research-labs/tap/sfae
npm install -g @fundamental-research-labs/sfae
curl -fsSL https://sfae.io/install.sh | sh

How it works

  1. Check

    Agent reads the service's official API/auth docs, then checks what credentials exist for a domain.

    sfae credentials github.com
  2. Prompt

    Human provides what's missing via a browser form. The agent keeps this command running and waits until it exits; credential collection is human-paced and should have no agent-side timeout.

    sfae prompt github.com --spec '{
  "help_url": "https://github.com/settings/tokens",
  "fields": ["ACCESS_TOKEN"]
}'
  3. Code

    If a workflow asks for a short-lived 2FA/MFA code, SFAE opens a transient browser form and prints the code for immediate submission without storing it.

    sfae code github.com --label Work --length 6
  4. Request

    Agent makes calls with placeholders. SFAE resolves them.

    sfae request GET "https://api.github.com/user" \
  -H "Authorization: Bearer {ACCESS_TOKEN}" \
  -H "User-Agent: sfae"
